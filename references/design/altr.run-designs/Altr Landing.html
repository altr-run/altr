Chapter 02 · The Loop

Idea to shipped, in one room.

No "sync to Linear." No "export to Figma." One graph. Five steps. Your hands stay on the keyboard.

idea → spec → tickets
→ code → PR

You

02

A working draft appears below your intent. Edit any line; Pax adapts the rest in real time.

03

The spec decomposes into a dependency-aware board. You flag each one: autopilot, copilot, or human-only.

This should feel like Figma multiplayer, not ChatGPT-in-a-sidebar. Pax's cursor, your cursor, both leaving tracks. The edits interleave. The spec gets better with every pass.

Specs / Auth / magic-link ● v3 · live

The 30-day session.

A returning user on a trusted device shouldn't hit login every 7 days. That's a security ritual, not a UX decision.

Rotate the session token on every use — the reuse-on-sight pattern. If a token is seen twice, one of the two is stolen; we log both out and ask a question only the real user can answer

Chapter 03 · The Bet

Two beliefs, baked into the product.

Altr is only useful if these two things are true. They are. The rest is craft.

Belief 01

The 3-person team is the new 30-person team.

In the next 24 months, small teams with agent leverage will out-ship traditional product orgs by 5–10×. The constraint stops being headcount. It starts being direction. Altr is built for the people who plan to be fewer, not more.

Belief 02

Integration cost is the bottleneck, not AI capability.

You don't need a better individual tool. You need one surface where PM, design, eng, and research — humans and agents — read and write the same data. Linear will add AI features. Altr is AI teammates as the substrate.

Chapter 06 · Questions

Things people ask before they trust us.

Short, honest answers. If something isn't here, write to hello@altr.run.

Do I need to use Claude Code?
For v0.1, yes. Eng runs on the Claude Code CLI under the hood. You bring an Anthropic API key, we spawn the subprocess, you own the output. OpenAI Codex support is planned for v0.3.
What does "autopilot" actually ship?
An autopilot ticket runs end-to-end: Eng reads the spec, writes code in an isolated worktree, commits, opens a PR on GitHub, and posts the diff back into Altr for your review. You never lose the right to say no — nothing merges without your approval.
Does my code leave my machine?
No. Altr is local-first — SQLite on disk, git operations on your filesystem, subprocess execution in your shell. Only the prompts and context you explicitly send to the LLM ever leave.
Is this a coding agent, or something bigger?
Something bigger — deliberately. Better coding agents are a crowded fight. Altr's bet is on the whole room: PM, eng, design, research, all co-habiting the same artifacts. Coding happens to be where v0.1 begins because that's where the pain is sharpest.
Can I use this alone?
Altr is built solo-first. One human, two agents, one Mac. Team mode comes later (v0.5, month 10–12) via CRDT. The single-user experience is the one we're obsessing over first.
Why a Mac app, not a web tool?
Subprocess control, file watching, git access, keyboard latency, offline work. Web apps give you none of these without great pain. And an app you can feel — cold-launches in 300ms, uses 200MB — is a craft signal you can't fake.
What about my data?
Your specs, tickets, conversations, and artifacts live in a local SQLite database. We do not have a server that stores them. When team mode ships, sync is opt-in and end-to-end encrypted.
How much does this really cost per month?
Altr: $29/mo Solo. LLM: whatever you decide — we recommend $50–200/mo in Anthropic spend for active shipping. Total for a serious solo builder: ~$100–250/mo. A fraction of one headcount.
