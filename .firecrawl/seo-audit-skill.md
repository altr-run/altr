# SEO Audit

[Permalink: SEO Audit](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#seo-audit)

You are an expert in search engine optimization. Your goal is to identify SEO issues and provide actionable recommendations to improve organic search performance.

## Initial Assessment

[Permalink: Initial Assessment](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#initial-assessment)

**Check for product marketing context first:**
If `.agents/product-marketing-context.md` exists (or `.claude/product-marketing-context.md` in older setups), read it before asking questions. Use that context and only ask for information not already covered or specific to this task.

Before auditing, understand:

1. **Site Context**
   - What type of site? (SaaS, e-commerce, blog, etc.)
   - What's the primary business goal for SEO?
   - What keywords/topics are priorities?
2. **Current State**
   - Any known issues or concerns?
   - Current organic traffic level?
   - Recent changes or migrations?
3. **Scope**
   - Full site audit or specific pages?
   - Technical + on-page, or one focus area?
   - Access to Search Console / analytics?

## Audit Framework

[Permalink: Audit Framework](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#audit-framework)

### Schema Markup Detection Limitation

[Permalink: Schema Markup Detection Limitation](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#schema-markup-detection-limitation)

**`web_fetch` and `curl` cannot reliably detect structured data / schema markup.**

Many CMS plugins (AIOSEO, Yoast, RankMath) inject JSON-LD via client-side JavaScript — it won't appear in static HTML or `web_fetch` output (which strips `<script>` tags during conversion).

**To accurately check for schema markup, use one of these methods:**

1. **Browser tool** — render the page and run: `document.querySelectorAll('script[type="application/ld+json"]')`
2. **Google Rich Results Test** — [https://search.google.com/test/rich-results](https://search.google.com/test/rich-results)
3. **Screaming Frog export** — if the client provides one, use it (SF renders JavaScript)

Reporting "no schema found" based solely on `web_fetch` or `curl` leads to false audit findings — these tools can't see JS-injected schema.

### Priority Order

[Permalink: Priority Order](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#priority-order)

1. **Crawlability & Indexation** (can Google find and index it?)
2. **Technical Foundations** (is the site fast and functional?)
3. **On-Page Optimization** (is content optimized?)
4. **Content Quality** (does it deserve to rank?)
5. **Authority & Links** (does it have credibility?)

## Technical SEO Audit

[Permalink: Technical SEO Audit](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#technical-seo-audit)

### Crawlability

[Permalink: Crawlability](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#crawlability)

**Robots.txt**

- Check for unintentional blocks
- Verify important pages allowed
- Check sitemap reference

**XML Sitemap**

- Exists and accessible
- Submitted to Search Console
- Contains only canonical, indexable URLs
- Updated regularly
- Proper formatting

**Site Architecture**

- Important pages within 3 clicks of homepage
- Logical hierarchy
- Internal linking structure
- No orphan pages

**Crawl Budget Issues** (for large sites)

- Parameterized URLs under control
- Faceted navigation handled properly
- Infinite scroll with pagination fallback
- Session IDs not in URLs

### Indexation

[Permalink: Indexation](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#indexation)

**Index Status**

- site:domain.com check
- Search Console coverage report
- Compare indexed vs. expected

**Indexation Issues**

- Noindex tags on important pages
- Canonicals pointing wrong direction
- Redirect chains/loops
- Soft 404s
- Duplicate content without canonicals

**Canonicalization**

- All pages have canonical tags
- Self-referencing canonicals on unique pages
- HTTP → HTTPS canonicals
- www vs. non-www consistency
- Trailing slash consistency

### Site Speed & Core Web Vitals

[Permalink: Site Speed & Core Web Vitals](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#site-speed--core-web-vitals)

**Core Web Vitals**

- LCP (Largest Contentful Paint): < 2.5s
- INP (Interaction to Next Paint): < 200ms
- CLS (Cumulative Layout Shift): < 0.1

**Speed Factors**

- Server response time (TTFB)
- Image optimization
- JavaScript execution
- CSS delivery
- Caching headers
- CDN usage
- Font loading

**Tools**

- PageSpeed Insights
- WebPageTest
- Chrome DevTools
- Search Console Core Web Vitals report

### Mobile-Friendliness

[Permalink: Mobile-Friendliness](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#mobile-friendliness)

- Responsive design (not separate m. site)
- Tap target sizes
- Viewport configured
- No horizontal scroll
- Same content as desktop
- Mobile-first indexing readiness

### Security & HTTPS

[Permalink: Security & HTTPS](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#security--https)

- HTTPS across entire site
- Valid SSL certificate
- No mixed content
- HTTP → HTTPS redirects
- HSTS header (bonus)

### URL Structure

[Permalink: URL Structure](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#url-structure)

- Readable, descriptive URLs
- Keywords in URLs where natural
- Consistent structure
- No unnecessary parameters
- Lowercase and hyphen-separated

## On-Page SEO Audit

[Permalink: On-Page SEO Audit](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#on-page-seo-audit)

### Title Tags

[Permalink: Title Tags](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#title-tags)

**Check for:**

- Unique titles for each page
- Primary keyword near beginning
- 50-60 characters (visible in SERP)
- Compelling and click-worthy
- Brand name placement (end, usually)

**Common issues:**

- Duplicate titles
- Too long (truncated)
- Too short (wasted opportunity)
- Keyword stuffing
- Missing entirely

### Meta Descriptions

[Permalink: Meta Descriptions](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#meta-descriptions)

**Check for:**

- Unique descriptions per page
- 150-160 characters
- Includes primary keyword
- Clear value proposition
- Call to action

**Common issues:**

- Duplicate descriptions
- Auto-generated garbage
- Too long/short
- No compelling reason to click

### Heading Structure

[Permalink: Heading Structure](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#heading-structure)

**Check for:**

- One H1 per page
- H1 contains primary keyword
- Logical hierarchy (H1 → H2 → H3)
- Headings describe content
- Not just for styling

**Common issues:**

- Multiple H1s
- Skip levels (H1 → H3)
- Headings used for styling only
- No H1 on page

### Content Optimization

[Permalink: Content Optimization](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#content-optimization)

**Primary Page Content**

- Keyword in first 100 words
- Related keywords naturally used
- Sufficient depth/length for topic
- Answers search intent
- Better than competitors

**Thin Content Issues**

- Pages with little unique content
- Tag/category pages with no value
- Doorway pages
- Duplicate or near-duplicate content

### Image Optimization

[Permalink: Image Optimization](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#image-optimization)

**Check for:**

- Descriptive file names
- Alt text on all images
- Alt text describes image
- Compressed file sizes
- Modern formats (WebP)
- Lazy loading implemented
- Responsive images

### Internal Linking

[Permalink: Internal Linking](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#internal-linking)

**Check for:**

- Important pages well-linked
- Descriptive anchor text
- Logical link relationships
- No broken internal links
- Reasonable link count per page

**Common issues:**

- Orphan pages (no internal links)
- Over-optimized anchor text
- Important pages buried
- Excessive footer/sidebar links

### Keyword Targeting

[Permalink: Keyword Targeting](https://github.com/zhangzhang-111-i/marketingskills/blob/main/skills/seo-audit/SKILL.md#keyword-targeting)

**Per Page**

- Clear primary keyword target
- Title, H1, URL aligned
- Content satisfies search intent
- Not competing with other pages (cannibalization)

**Site-Wide**

- Keyword mapping document
- No major gaps in coverage
- No keyword cannibalization
- Logical topical clusters
